This article covers the installation and configuration of Arch Linux on a Lenovo ThinkPad T14/T14s (Intel) Gen 1 laptop. Everything seems to work pretty much out the box with kernel >=5.9.0. Yet untested: wwan, smartcard reader, bluetooth.

Firmware

Secure boot

As of January 2021 deleting SecureBoot keys and installing your own keys (for example by using KeyTool) will brick the device. This is a problem that is similar to one which has been reported on some other Lenovo laptops and is likely due to a faulty firmware. If the device is stuck in a boot loop after replacing the SecureBoot keys, the only way to repair it is by replacing the mainboard of the device. Hopefully, the issue will get fixed with a firmware update in the future.

Power modes

Warning: BEWARE OF POSSIBLE SKIN BURNS, THE DEVICE GOES BEYOND 75 C IN POWER MODE. Some Thinkpad laptops has a "lap mode" detection, used to throttle the device if it detects that you are using it in your lap, which isn't something that is implemented for Linux, there isn't anything that will detect that the laptop is in your lap and throttle the device. See [1]

Lenovo Vantage software manages 3 power modes in Windows with Intel DPTF, they are managed through acpi calls:

  • economy
  • balanced
  • performance

There isn't a software to manage Intel DPTF for this laptop, so we need to do that manually.

If you don't enable the performance mode, the GPU will throttle at 57 C and the CPU Mhz will be throttled too. Which makes applications stutter. So far, in a desk, using the performance mode while connected to the power supply wasn't a problem.

For the ThinkPad T14 Intel, the ACPI calls are as follows:

economy \_SB.PCI0.LPCB.EC._Q6E
balanced \_SB.PCI0.LPCB.EC._Q6F
performance \_SB.PCI0.LPCB.EC._Q6D

To enable power mode, install the acpi package and issue the following commands in bash: 

c(){ echo "$1" | sudo tee /proc/acpi/call >/dev/null && sudo cat /proc/acpi/call;echo;}
c "\_SB.PCI0.LPCB.EC._Q6D"

Success outputs are like: 

0x8012b01

You can check what is the throttling temperature in Celsius for the NVIDIA GPU if you want to make sure, note that you need the nvidia-settings package installed: 

nvidia-settings -q GPUMaxOperatingTempThreshold

An output of 87C or 75C means it is in performance mode.

NVIDIA

The opensource NVIDIA drivers work correctly with no special modification. To use it's full power, see Lenovo ThinkPad T14/T14s (Intel) Gen 1#Power modes
