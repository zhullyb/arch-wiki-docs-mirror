Arch Linux 係一個獨立開發、 x86-64 通用嘅 GNU/Linux 發行版。 佢主要着重喺用漸進式發行嘅模式去畀用戶各種最新穩定版本嘅軟件。 預設安裝係一個最細嘅基本系統，用家可以自行更改設定同埋加所需嘅嘢。

原則

簡單樸素

Arch Linux 定義「簡單樸素」(simplicity) 為「冇唔需要嘅添加同改動」（without unnecessary additions or modifications）。 佢提供喺下游發行版特定，盡可能少地改進咗，上游 (upstream)開發人源發布嘅軟件： 上游唔接納嘅修補程式 (patches)會被避免，而 Arch 嘅下游修補程式幾乎都係為咗向後移植 (backporting)，喺下一次嘅發布會畀人廢棄。

以類似方式， Arch 提供上游畀嘅、可能因為發行版嘅特定問題（好似更改系統檔案路徑噉）而有些少改動嘅設定檔案。 佢唔會因為個軟體包裝咗而加上自動化嘅功能——如啟用一個單元。軟體包只會因為好似慳番儲存容量等大好處而拆開啲軟體包。 圖像用家介面嘅設定工具唔會正式被提供，以鼓勵用家用個 shell 同文字編輯器去更改大部分嘅系統設定。

現代化

Arch Linux 只要可以合理噉避免系統嘅軟體包破損，就會着重喺維持住自己最新穩定嘅軟件。佢係基於一個漸進式發行嘅系統，畀用家裝一次而又可以連續升級。

Arch 有好多畀 GNU/Linux 用戶、比較新嘅功能，好似 systemd 初始化系統，現代 文件系统, LVM2, 軟件 RAID, udev 支持同埋 initcpio (同 mkinitcpio)，仲有最新嘅內核。

實用主義

Arch 係一個講求實用嘅發行版。呢度嘅原則只係有用嘅方針。話哂，設計方面嘅決定都係由開發人員同意之後先至作出。基於證據嘅技術性同辯論先至係最重要，而唔係政治同民意。

好多喺 Arch Linux 倉庫 (repository) 嘅軟體包同構建腳本都都係免費同開源(open-source)。倉庫仲有功能性高嘅閉源軟體包添。

用戶就係中心

同好多想要令自己更加容易使用 (user-friendly) 嘅 GNU/Linux 發行版唔同，Arch Linux 直到永遠都係以用戶為中心 (user-centric)。呢個發行版希望能夠滿足貢獻者嘅需求，並唔係去吸引盡可能多嘅使用者。佢專係畀一啲對 GNU/Linux 入木三分、瞭如指掌嘅使用者，同埋任何有「自己郩手」(do-it-yourself)態度嘅人——即係願意去睇文檔同自行解法問題嘅人。

全部使用者都被鼓勵去參與同貢獻呢個發行版。 報告同幫手修理漏洞係一件好值得去重視嘅事；改進軟體包或核心計劃嘅修補程式係一件好值得讚賞嘅事： Arch 嘅開發人員都係志願者，活躍貢獻者通常都會發覺自己成為咗團隊嘅一部分。 Archers 可以自由噉貢獻軟體包到 Arch User Repository；對 ArchWiki 嘅文檔進行改進；喺論壇度為其他人提供技術上嘅支援；或者齋去個論壇、郵件列表、或[[IRC channels]度進行交流，切磋琢磨，表達下自己嘅意見。 Arch Linux 係世界上好多唔同人嘅作業系統之選，而天下有好幾個喺唔同語言下提供幫助同文檔嘅國際社區。 正所謂「施比受更為有褔」，話哂你自己都用緊 Arch Linux，噉自己都會受益啦。

一物多用

麻雀雖小，五臟俱全。Arch Linux 係一個多功能嘅發行版。裝咗之後，佢淨係得一個命令行介面，並唔會有多餘、冇用嘅軟體包。 用家可以自行喺畀x86-64嘅官方倉庫入面數以萬計嘅高質素軟體包揀其中一啲，噉就可以建立一個啱自己嘅系統。

Arch 用pacman，一個輕便嘅軟體包管理員，淨係需要一個命令就可以升級成個系統，簡單方便快捷。 Arch 仲有一個好似 ports 嘅包構建系統－－Arch Build System，令到由開源碼建立到裝嘢都乾脆俐落，亦都淨係需要行個命令就可以進行同步，簡直係一箭中的、一氣呵成。 除此之外，Arch User Repository 有無數多由貢獻者供給嘅 PKGBUILD 腳本可從原始碼配合 makepkg 作編譯可以裝嘅軟體包。使用者亦可以輕而易舉地建立同維謢佢哋自己嘅倉庫。

歷史

Arch 嘅社區已經成長同成熟咗，成為最受歡迎同影響最深遠嘅 Linux 發行版之一，每年都受到廣大評價同注意。

Arch 嘅開發人員係兼職義工，並唔收費。亦都因為 Arch 嘅營利前景並唔樂觀，所以佢維持自由，直到永遠。有興趣想睇更多關於 Arch 嘅開發歷史嘅話可以睇下互聯網回溯紀錄嘅 Arch 條目同埋Arch Linux 嘅新聞條目。

背景

Judd Vinet 係一個加拿大人同一個兼職吉他手，喺2001年開始開發 Arch Linux。佢嘅第一個正式版本—— Arch Linux 0.1 ——喺2002年3月11號發布。佢畀Slackware、BSD、PLD Linux 同 CRUX 的優雅簡潔所啟發，但係當時他們缺乏完善嘅軟體包管理系統感到失望。於是，Vinet 根據嗰啲發行版相似嘅原則整咗自己嘅發行版。但係，他自己都開發咗自己嘅軟體包管理系統Pacman，自動執行軟體包嘅安裝、刪除同升級。

早年

早期嘅 Arch 社區發展穩定(論壇帖子，用戶和錯誤報告的圖表)。Arch 社區早期就已經畀人嗌做一個開放、友好同有幫助嘅社區。

ArchWiki 嘅誕生

ArchWiki 喺2005年7月8號用 MediaWiki 起。

A. Griffin 時代

喺2007年年尾，Judd Vinet 唔再積極參與 Arch 嘅開發，並且順利將 Arch 嘅開發工作轉移畀美國程序員 Aaron Griffin，即係 Phrakture。

Arch 嘅安裝腳本

喺2012年7月15號發布嘅安裝鏡像廢除咗菜單驅動嘅Arch 安裝框架(AIF)，並以Arch 安裝腳本 (arch-install-scripts)取而代之。

systemd 時代

喺 2012 至 2013 年，傳統嘅 System V init system 畀 systemd 取代。[1][2][3][4]

淘汰i686

喺2017年1月25號， Arch 宣布因為用 i686 架構嘅開發者同社區人數下降，於是停止對 i686 架構嘅支持。喺2017年11月尾，所有喺鏡像度嘅 i686 軟體包都畀人剷曬。

領導人選舉

2020年年頭，喺團隊嘅努力下， Arch Linux 嘅員工設計咗一套新嘅程序確定未來嘅領導人，記錄咗喺DeveloperWiki:Project Leader。

由於 Aaron Griffin 決定退位，於是揀咗個新領導人取而代之。喺2020年2月24號，結果出爐，令 Levente Polyak 當選為正式嘅領導人。
