Pre-installation actions

  • Disable Secure Boot in your BIOS (press the F2 key on boot).

Installation notes

Install Arch Linux as usual using UEFI installation method. Use a recent kernel (>=5.8.x), otherwise arch will have trouble booting up. Some kernels also need you to switch back and forth tty (ctrl+alt+f2 and ctrl+alt+f1) to make the login page appear. You may have to use the amdgpu.runpm=0 Kernel parameters if you are experiencing GPU crashes.

BIOS updating, GPU firmware updating

Use windows to update BIOS and GPU firmware. BIOS can be updated through the BIOS though.

Graphics

Works out of the box, install the mesa if you want to use 3D applications. If you want to use the dGPU, use $ DRI_PRIME=1 command , where command is the command you want to launch with your dGPU (see PRIME#For open source drivers - PRIME). Wayland session works out of the box on gnome with pipewire.

External monitor

Works out of the box. The USB-c port carry a display port signal.

Temperature monitoring, Fan control

Monitoring

By default the kernel loads the k10temp module to check cpu thermals. 

$ sensors
 k10temp-pci-00c3
 Adapter: PCI adapter
 Tctl:         +38.0°C  
 Tdie:         +38.0°C

To monitor GPU temp, and see fan speeds you will have to force load the dell-smm-hwmon Kernel module (see [2] for documentation), which is not loaded by default on this laptop. 

$ sudo modprobe dell-smm-hwmon restricted=0 ignore_dmi=1

To make this setting persist upon reboot edit your /etc/modules-load.d/dell-smm-hwmon.conf file 

dell-smm-hwmon

and your /etc/modprobe.d/dell-smm-hwmon.conf file 

# This file must be at /etc/modprobe.d/
options dell-smm-hwmon restricted=0 ignore_dmi=1

You should now see a dell result in the result of sensors. 

dell_smm-virtual-0
Adapter: Virtual device
Processor Fan:    0 RPM
Video Fan:        0 RPM
Other:          +37.0°C  
CPU:            +40.0°C  
Ambient:        +38.0°C  
GPU:            +38.0°C  
Other:          +37.0°C

Scripts to control fans

The following script will allow you to control fans. You can save it to /usr/bin/adjust_fans.sh (for instance). 

set_to=0
re='^[0-9]+$'
if ! "$1" =~ $re  ; then
	status=($(<"/etc/fan_status"))
	set_to=128
if [ "$status" -ne 255 ]; then
	set_to=255
fi
else
	set_to=$1
fi
sudo su -c 'tee /sys/class/hwmon/hwmon4/pwm{1,3} /etc/fan_status' 2> /dev/null << EOF
$set_to
EOF
printf "Set both fans to %s\n" "$set_to"

Edit the hwmon4 in the script according to your result of the following command. 

$ grep dell_smm /sys/class/hwmon/hwmon*/name
/sys/class/hwmon/hwmon4/name:dell_smm

Allow this script to be executable 

$ chmod +x /usr/bin/adjust_fans.sh

If you don't want to type your password every time you use it, you can add this line to your /etc/sudoers file. 

ALL ALL=(root) NOPASSWD: /usr/bin/adjust_fans.sh

You should now be able to control the fans of your laptop with the commands 

$ sudo adjust_fans.sh

or 

$ sudo adjust_fans.sh speed

where speed is included in the set {0,128,256} (0, half speed, full speed). You can also map this command to a keyboard shortcut (via your DE settings).

Thermal management, overclocking

CPU

The packages ryzenadj-gitAUR and ryzen-controller-binAUR (gui for ryzenadj) should work out of the box to control maximum temperature and TDP of your CPU. For instance, the following command restrict your CPU TDP to 40 Watts and maximum temperature to 70°C (perfectly safe on this laptop) 

$ sudo ryzenadj -a 40000 -b 40000 -c 40000 -f 70
Warning: Don't touch anything that you don't understand, a too high TDP or temperature can permanently damage your CPU/iGPU !

see [4] and [5] for detailed instructions on how to use theses tools.

Undervolting and overclocking are not available on this laptop, ZenStatesAUR[broken link: package not found] does not seem to have any effect on zen2 processors see [6] .

Also see Improving performance for more performance tricks.

GPU

You can try to use corectrlAUR. Also see AMDGPU#Overclocking.
